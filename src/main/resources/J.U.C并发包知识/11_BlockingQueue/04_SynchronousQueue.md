## SynchronousQueue
```
SynchronousQueue是一个无容量的队列, 也就是说该队列不允许存放数据, 提取数据和放入数据的操作必须是
同时进行的, 一般用take和put进行配合操作, take在没有线程提供数据的情况下会进入阻塞, put在没有线程
拿取数据的情况下也会进入阻塞

增加方法:
  offer(T t): 如果正好有线程在调用该队列的take方法而陷入阻塞, 则会将元素传递给take方法, 并且返回
              true, 如果没有线程正在调用take方法而陷入阻塞, 则返回false, 该方法是一个不阻塞的方法,
              换句话说, 如果正好有人等待我给数据, 我就给这个数据给它, 如果没有, 我就不给了, 执行
              之后的代码
  put(T t): 等待其它线程调用poll, take, remove方法来获取数据, 如果没有则陷入阻塞直到有才会被唤醒
  add(T t): 如果有其它线程调用了take方法拿取数据失败而陷入阻塞, 那么该方法就会把数据交给这个take
            方法, 如果没有其它线程调用take方法获取数据, 那么该方法就会报错, 其内部就是调用了offer,
            如果offer返回false, 则报错

删除方法:
  take(): 等待其它线程调用offer, put, add方法来放入数据, 如果没有则陷入阻塞直到有才会被唤醒
  poll(): 如果有其它线程调用了put方法等待交换数据, 那么poll方法就会拿取这个数据, 如果没有则返回null

  remove(): 如果有其它线程调用了put方法等待交换数据, 那么remove方法就会拿取这个数据, 如果没有则报
            错, 其内部就是调用了poll方法, 如果poll方法返回null, 则报错

其它方法:
  remainingCapacity: 一直会返回0, 因为是一个没有容量的队列
  contains: 一直返回false
```
